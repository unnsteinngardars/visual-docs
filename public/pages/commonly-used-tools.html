Commonly Used Tools for Investigations

Starter toolkit for triage, static & dynamic analysis, and network review

These tools are the basic ones used for initial investigations. Click a tool name to visit its site, or expand a card for deeper usage notes.

Procmon
A helpful tool for tracking system activity, especially regarding malware research, troubleshooting, and forensic investigations.
More details

Process Monitor captures real-time file system, registry, process, thread, and network activity in one unified timeline. It's the go-to choice for watching what a suspect binary actually touches when it runs.

  • Filtering: narrow by process name, operation (CreateFile, RegSetValue), path, or result — keeps noise manageable.
  • Boot logging: capture activity before logon-time persistence kicks in.
  • Save as PML: share captures with another analyst or parse them with tooling.

Part of the Sysinternals Suite — runs portable, no install.

Process Explorer
Shows the parent-child relationship of processes, DLLs loaded, and their paths.
More details

Think of it as Task Manager on steroids. The tree view makes it trivial to spot a suspicious process launched by winword.exe or explorer.exe.

  • Handles / DLL view: toggle the lower pane to see every handle or loaded module per process.
  • VirusTotal integration: right-click → submit hashes to get reputation in-line.
  • Verified signers: column highlights unsigned/unknown binaries at a glance.

Combine with Procmon: identify the suspect PID here, then filter Procmon to it.

HxD
Malicious files can be examined or altered via hex editing.
More details

Fast Windows hex editor supporting large files, disk sectors, and live memory of running processes.

  • Magic bytes & headers: verify the true file type (e.g. 4D 5A for PE, 50 4B 03 04 for ZIP/OOXML).
  • Patching: flip bytes to neutralize a known-bad JMP or clear an AV-triggering string when defanging a sample.
  • Data inspector: instantly interpret bytes as int/float/GUID/timestamp.
Wireshark
Observing and investigating network traffic to look for unusual activity.
More details

The reference-grade packet capture and protocol dissector. Essential for spotting C2 beacons, data exfil, DNS tunneling, and unauthorised lateral movement.

  • Display filters: http.request, dns, tcp.stream eq 4, ip.addr == 10.0.0.5.
  • Follow TCP/TLS stream: reconstruct the full conversation instead of reading packet-by-packet.
  • Export objects: pull payloads / binaries out of HTTP, SMB, and FTP captures.

Pair with tcpdump on the wire, then analyse the .pcap here offline.

CFF Explorer
Can generate file hashes for integrity verification, authenticate the source of system files, and validate their validity.
More details

A PE (Portable Executable) editor by NTCore. Gives a structured view of every header, section, import, and resource inside a Windows binary — and lets you modify them.

  • Hash calculator: MD5/SHA-1/SHA-256 for IOC sharing and VT lookups.
  • Imports / exports: spot suspicious API usage like VirtualAllocEx, CreateRemoteThread.
  • Signature check: verify digital signatures and the signing chain.
  • Rebuild PE: fix headers after unpacking a dumped process.
PEStudio
Static analysis or studying executable file properties without running the files.
More details

Loads a Windows binary without executing it and surfaces indicators an analyst cares about, colour-coded by risk.

  • Blacklisted strings & APIs: anti-debug, injection, keylogging, crypto routines flagged automatically.
  • VirusTotal lookup: built-in hash submission shows engine hits without uploading the file itself.
  • Indicators tab: centralised summary of everything suspicious, ranked.

Great first-pass triage tool before opening the sample in a disassembler.

FLOSS
Extracts and de-obfuscates all strings from malware programs using advanced static analysis techniques.
More details

FLARE Obfuscated String Solver (by Mandiant) does what strings.exe can't: it automatically recovers strings that malware builds at runtime on the stack or decodes with simple ciphers.

  • Static strings: traditional ASCII + UTF-16 extraction.
  • Stack strings: recovered from instructions that construct strings character-by-character.
  • Decoded strings: emulator-assisted recovery of XOR/ROT/RC4-obfuscated content.

Run early — recovered URLs, mutexes, and registry keys often reveal the malware family instantly.

Starter investigation kit · Verify each binary's authenticity before running · Detonate samples only inside an isolated VM
