📜01 · Chain of Custody
Document exhibit ID, acquisition date/time (UTC + local), examiner, witness
Photograph scene, serials, screen state BEFORE touching
Record every transfer — custodian, purpose, hashes at each handoff
Hash source and image (MD5 + SHA-256) — verify match before analysis
Use write-blocker (hardware preferred) for any physical media
Store originals read-only; work exclusively from verified copies
⏱02 · Order of Volatility
CPU registers, cache, ARP & routing tables (seconds)
RAM — processes, handles, network sockets, keys (minutes)
Temp files, pagefile.sys / swapfile.sys, hibernation (minutes–hours)
Disk: allocated files, MFT, journals, event logs (persistent)
Remote logs, backups, cloud/SIEM retention (long term)
Physical printouts, archived media (long term)
💽03 · Disk Imaging
dd raw image + hash
dd if=/dev/sdb of=case.raw bs=4M conv=noerror,sync status=progress
dcfldd (dd fork with hashing)
dcfldd if=/dev/sdb hash=sha256 hashlog=hash.txt of=case.raw
FTK Imager CLI (E01)
ftkimager /dev/sdb case --e01 --compress 6 --case-number 001
ewfacquire (EnCase E01)
ewfacquire -t case -c best -d sha256 /dev/sdb
Verify image integrity
sha256sum case.raw; ewfverify case.E01
Mount read-only for analysis
mount -o ro,loop,noexec,noatime case.raw /mnt/evidence

🚨Anti-Forensics Red Flags
⚠
$SI timestamps earlier than $FN — classic timestomp signature
⚠
EID 1102 / 104 — Security or System log cleared
⚠
EID 4616 — system time changed mid-activity
⚠
USN entries for files no longer on disk — secure deletion
⚡
sdelete / cipher /w / BleachBit presence in prefetch
⚡
VSS deletion (vssadmin delete shadows) immediately before incident
⚡
Unusually uniform cluster entropy — wipe/overwrite pattern
